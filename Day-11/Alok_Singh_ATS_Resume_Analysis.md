## 7. Key Learnings

1. **ATS scores are capped by real skill gaps, not just wording.** No amount of rephrasing can make a resume match keywords for technologies you haven't actually used — React.js and MongoDB here are the clearest example.
2. **Implied skills should be stated explicitly.** Using Express.js almost always means working with REST APIs and Git, but ATS systems match literal keywords, not inferred ones — so name them directly if true.
3. **Section naming matters.** "Professional Summary" scores better than "Career Objective" in most modern ATS parsers, even with similar content.
4. **Formatting simplicity wins.** Single-column layouts, standard fonts, and standard section headers (Experience, Education, Skills) parse far more reliably than tables, columns, or graphics.
5. **Tailor per job, don't mass-apply one resume.** Since keyword weighting shifts with each JD, the highest-scoring resume is always the one re-optimized for that specific posting — as done here.
