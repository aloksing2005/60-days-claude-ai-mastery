# Day 7 – Claude Model Selection & Reasoning Effort

## Theme
Claude Fundamentals

## Objective
Learn how to choose the right Claude model and reasoning effort for different tasks.

---

## My Claude Usage Strategy

### Recommended Primary Model

**Claude Sonnet 5**

### Why This Model Fits Me

- Best balance of speed and intelligence
- Excellent for coding and debugging
- Great for learning AI concepts
- Ideal for Prompt Engineering
- Perfect for Full Stack Development projects

---

## When to Use Haiku

- Quick summaries
- Syntax checks
- Small coding tasks
- Simple questions
- Fast responses

---

## When to Use Sonnet

- Daily coding
- Debugging
- Web Development
- Flutter projects
- AI learning
- Prompt Engineering
- Resume improvement

---

## When to Use Opus

- Deep research
- Complex debugging
- System design
- Architecture planning
- AI strategy

---

## Recommended Reasoning Effort

| Effort | Use Case |
|---------|----------|
| Low | Simple lookups |
| Standard | Daily coding & learning |
| High | Debugging & architecture |
| Max | Research & complex planning |

---

## Personalized Workflow

| Task | Model | Effort | Reason |
|------|--------|--------|--------|
| Coding | Sonnet 5 | Standard | Best balance |
| Debugging | Sonnet 5 | High | Better reasoning |
| Learning | Sonnet 5 | Standard | Clear explanations |
| Resume | Sonnet 5 | High | Better optimization |
| Research | Opus 4.8 | Max | Deep analysis |
| Quick Questions | Haiku 4.5 | Low | Fast answers |

---

## Biggest Learnings

- Choosing the correct model improves output quality.
- Sonnet is the best daily model.
- Haiku is ideal for quick tasks.
- Opus should be used for complex reasoning.
- Reasoning effort should match task complexity.
