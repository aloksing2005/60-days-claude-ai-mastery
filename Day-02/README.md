# Day 2 – Prompt Engineering

## Theme
Phase 1 – Foundation & Prompt Engineering

## Objective
Understand the difference between a normal prompt and an engineered prompt.

## Weak Prompt

Make a Hospital Management System.

## Engineered Prompt

Act as a Senior Full Stack Software Architect.

I am a Computer Science student building a Hospital Management System using Node.js, Express.js, MongoDB, and React.

Design the following modules:

- Patient Management
- Doctor Management
- Appointment Scheduling
- Billing
- Admin Dashboard

For each module provide:
- Database Schema
- REST API Endpoints
- Business Logic
- Security Considerations

Use role-based authentication and explain everything step by step.

## Key Learnings

- A normal prompt is vague and lacks context.
- An engineered prompt provides clear goals, context, constraints, and expected output.
- Better prompts produce more accurate and useful AI responses.
- Prompt Engineering reduces ambiguity and improves AI collaboration.
