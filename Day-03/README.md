# Day 3 – Role-Based Prompting

## Theme
Phase 1 – Foundation & Prompt Engineering

## Objective
Learn how Role-Based Prompting transforms Claude from a general AI assistant into a specialized expert.

## What is Role-Based Prompting?

Role-Based Prompting means assigning a specific role or persona to the AI before asking a question. This helps generate more focused, accurate, and context-aware responses.

## Personas Compared

### Without Role
General AI Assistant

### Role 1
Senior Full Stack Software Architect

### Role 2
AI Product Manager

## Comparison

| Prompt Type | Result |
|--------------|--------|
| General Prompt | Generic response |
| Software Architect | Technical architecture, APIs, database design, scalability |
| AI Product Manager | User experience, AI features, business value, product strategy |

## Key Learnings

- Different roles provide different perspectives.
- Expert personas improve response quality.
- Role-based prompting makes AI outputs more relevant.
- Better prompts lead to better solutions.
- Personas help solve real-world software problems effectively.
